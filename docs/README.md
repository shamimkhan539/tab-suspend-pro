## 📝 Documentation Guidelines

When working with documentation:

1. **Always read from `docs/`** - All feature documentation lives here
2. **Add new docs to `docs/`** - Never create `.md` files in project root (except README.md, CHANGELOG.md, LICENSE)
3. **Update existing docs** - Keep documentation up-to-date in this folder
4. **Delete obsolete docs** - Remove outdated files to maintain clarity

### Folder Organization

-   `features/` - Feature-specific documentation organized by feature name
-   `guides/` - User-facing tutorials and guides
-   `development/` - Technical and development documentation
-   `api/` - API references and technical specifications

## 🔄 Maintenance

Documentation is maintained alongside code changes. When modifying features:

1. Update relevant documentation in `docs/features/`
2. Add user-facing guides to `docs/guides/` if needed
3. Document technical decisions in `docs/development/`
4. Update this index if adding new major documentation sections
